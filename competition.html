01 · 要解决什么问题

普通人写 SCI 论文的三座大山

🧱

结构壁垒：不知道每部分写什么

SCI 论文要求严格遵守 IMRaD 结构，但普通人打开空白文档后完全不知道该写什么、每一段应该覆盖哪些要素、以什么顺序组织论证。Introduction 需要完成"建立领域→识别缺口→陈述目的"三步论证，Discussion 需要完成"总结→对比文献→解释→局限→意义"五步推演——这些是学术界默认的"隐知识"，普通人无从获取。

🗣️

语言壁垒：不会写"学术英语"

学术英语是一套独立的修辞系统。"a significant increase was observed"和"things went up"表达的是同一事实，但在 SCI 审稿人眼中只有前者合格。问题不在于语法，而在于不知道学术写作有哪些表达范式——比如表达"前人研究有缺口"这个意图有几种地道写法？普通人不是能力不足，是选项库为零。

📐

工具壁垒：搞不定 LaTeX 和期刊格式

学术出版标准工具是 LaTeX，学习曲线陡峭。不同期刊（IEEE、Elsevier、Springer、Nature）有不同的字体、行距、双栏/单栏、图表规范、引文格式要求。手动切换和管理图表编号、交叉引用、参考文献格式极易出错，且对非计算机背景的人来说几乎不可能。

🔑 设计哲学

本系统的核心理念是"不替代用户思考，而是提供结构和选项"。我们拒绝使用任何 AI 文本生成技术——用户始终是自己论文的唯一作者。系统提供三层辅助：（1）结构引导（拆解好的子步骤表单）、（2）语言选项（可浏览和填充的学术句型模板库）、（3）格式自动化（填写完成后的一键排版导出）。每一层的输出都是确定性的——相同输入产生相同输出。

# Run 反序列化工厂
_RUN_TYPE_MAP = {
    "text": TextRun,
    "citation": CitationRun,
    "figure_ref": FigureRef,

def run_from_dict(d):
    run_type = d["type"]
    return _RUN_TYPE_MAP[run_type].from_dict(d)

06 · 核心技术模块

六大引擎详解

1. BibTeX/RIS 手写解析器

用正则表达式从零实现，零外部解析器依赖。支持括号深度追踪算法处理嵌套花括号值、作者名双格式自动识别（"Smith, John" / "John Smith"）、姓氏粒子检测（van/der/von）、后缀识别（Jr./III）、初始化规范化（"JK" → "J. K."）。附带格式自动检测功能。

2. 引文格式化引擎

五种风格独立实现：IEEE（方括号编号 + 缩写名）、APA 7th（作者-年份 + 斜体期刊 + DOI链接）、Vancouver（上标编号 + 无点缩写）、Nature（紧凑上标 + 逗号分隔）、MLA 9th（作者-页码体例）。每种风格有独立的作者名规范化函数。

3. LaTeX 编译器

DocumentAST → 完整 .tex 文件。内置 IEEE/Elsevier/Springer/Nature 四种 preamble 模板。自动处理 LaTeX 特殊字符转义（10 个映射）。逐段编译——TextRun → 转义文本，CitationRun → \cite{}，FigureRef → Fig.\ref{}。双次 xelatex 编译生成 PDF（解析交叉引用）。

4. Word 编译器

AST → python-docx → .docx。支持格式规则配置：字体族、字号、行距（单倍/1.5/双倍）、页边距（cm/in）、标题对齐。交叉引用在 Word 中转换为数字格式。图片使用 add_picture()，表格使用 add_table()。

5. 自动编号引擎

支持 Sequential（Fig. 1, 2, 3...全文连续）和 Chapter-based（Fig. 2.1, 2.2...按章节）两种模式。assign_numbers() 遍历元素按章节排序后分配编号。generate_labels() 生成 Fig. 1 / Table 2 等显示标签。

6. 声明式校验引擎

12 条规则以 JSON 对象数组存储——可扩展至数据库无需改代码。支持 6 种检查类型：required_field / required_fields / word_count_range / min_citations / citation_refs_match / figure_refs_exist。三级触发：字段失焦 / 步骤完成 / 导出前。
