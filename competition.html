01 · 问题分析

普通人写 SCI 论文的三座大山

学术论文写作被认为是"隐知识"最密集的人类活动之一。我们对 SCI 论文写作的困难做了系统分析，将其归纳为三个层级的问题，并针对每一层级设计了纯软件解决方案。

🧱

结构层：IMRaD 结构的认知鸿沟

SCI 论文要求严格遵守 IMRaD（Introduction, Methods, Results, and Discussion）结构，但普通人不知道每一部分应该写什么、写多少、以什么逻辑组织。 例如：Introduction 需要完成"建立领域→识别缺口→陈述目的→提出假设"四步论证链条， 而 Discussion 需要完成"总结发现→对比文献→解释机制→承认局限→引申意义"五步推演。

解决方案：6 步向导式表单将每步需完成的子任务拆解为填空题，每个输入框旁有上下文提示。

🗣️

语言层：学术英语的"方言壁垒"

学术英语有自己独特的措辞习惯和修辞范式——比如"a significant increase was observed"而非"we saw things go up"。 这需要长期浸淫学术环境才能习得。普通人不仅不知道某个句子该怎么写，甚至不知道"这个句子可以有哪些写法选项"。

解决方案：基于 Manchester Phrasebank 分类体系构建的句型模板库（30条核心模板），按写作功能和意图三维分类，用户通过下拉选择 + 变量填充即可生成地道学术表达。

📐

排版层：LaTeX 的技术壁垒与期刊适配

学术出版事实标准是 LaTeX，但非计算机背景的人学习 LaTeX 的成本极高。不同期刊（IEEE、Elsevier、Springer、Nature）有不同的模板、字体、行距、引文格式要求， 手动切换几乎不可能。图表编号、交叉引用、参考文献格式如果手动维护，极易出错。

解决方案：JSON → Document AST → LaTeX/Word 双编译管线，内置 4 种 SCI 期刊模板，图表公式自动编号，5 种引文格式一键切换。

🔑 设计哲学

本项目的核心理念是：不替代用户的思考，而是提供结构和选项。我们不使用任何文本生成技术——用户始终是自己论文的作者。 系统提供的是：（1）结构性的引导（每一步应该写什么）、（2）语言上的选项（学术句子可以怎么写）、 （3）格式上的自动化（写完后的排版由系统完成）。这三层分工明确，边界清晰。

上下文感知的帮助系统

每个文本框旁边有一个 [?] 图标，鼠标悬停时显示该段落在 SCI 论文中的功能说明、推荐长度、示范句型。 这不是简单的 tooltip——它根据当前章节动态加载不同的帮助文案。 Methods 部分会提示"请描述实验设计类型（如随机对照试验）"以及"充足到足以让其他研究者复现"的标准； Discussion 部分则会提示"解释你的结果意味着什么，而非重复结果本身"，并提醒至少引用 3 篇他人文献进行对比。

03 · 学术句型库 (Phrasebank)

动态模板引擎——不是文本生成，而是结构化选项系统

我们拒绝使用任何 AI 文本生成。每一条句型都是由人类手工编排的学术写作模板，基于 Manchester Academic Phrasebank 的功能分类体系。 用户通过选择功能意图、填充变量插槽，来完成"地道 SCI 英语"的拼接。

模板存储格式（数据库 JSONB）

{ "section": "introduction", "function_tag": "identifying_gap", "sub_function": "highlighting_problem", "template_text": "Despite considerable research on {existing_work}, {gap_description} remains poorly understood.", "slots": { "existing_work": { "type": "text", "required": true, "hint": "what has been studied" }, "gap_description": { "type": "text", "required": true, "hint": "what we still don't know" } }, "example_filled": "Despite considerable research on antibiotic resistance mechanisms, the role of environmental factors in resistance gene transfer remains poorly understood.", "academic_level": "intermediate", "source": "Manchester Phrasebank" }

# phrase_assembler.py — 核心类
class PhraseAssembler:
    SLOT_PATTERN = re.compile(r"\{(\w+)\}")

    @classmethod
    def fill(cls, template, slots_config, user_values, validate=True):
        # 1. 校验阶段：检查必填插槽 & 类型匹配
        if validate:
            cls._validate(template, slots_config, user_values)
        # 2. 替换阶段：逐一替换 {slot} 占位符
        result = template
        for slot_name in cls.extract_slots(template):
            value = user_values.get(slot_name, "")
            result = result.replace(
                f"{{slot_name}}", str(value))
        return result

    @classmethod
    def _validate(cls, template, slots_config, user_values):

编译流程（以 LaTeX 为例）

# latex_compiler.py
def compile_to_tex(self, ast: DocumentAST) -> str:
    parts = []
    parts.append(self._generate_preamble(ast))   # 导言区
    parts.append(r"\begin{document}")
    parts.append(self._generate_title_block(ast)) # 标题/作者
    if ast.abstract:
        parts.append(self._generate_abstract(ast.abstract))
    for section in ast.sections:
        parts.append(self._generate_section(section))
    for fig in ast.figures:
        parts.append(self._generate_figure(fig))
    for tbl in ast.tables:
        parts.append(self._generate_table(tbl))
    for eq in ast.equations:
        parts.append(self._generate_equation(eq))
    parts.append(self._generate_bibliography(ast))
    parts.append(r"\end{document}")
    return "\n".join(parts)

校验引擎：声明式规则 + 三层触发

12 条内置校验规则，覆盖 Methods / Results / Discussion / Abstract / Paper-level 五个维度。

规则定义示例

{ "rule_id": "methods_sample_size", "section": "methods", "trigger": "on_section_complete", "check": "required_field", "field": "sample_size", "severity": "error", "message": "请填写样本量（必须为正整数）", "auto_fix_hint": "在Methods → 参与者 → 样本量字段中填写" }

规则存储为 JSON 对象数组，可通过数据库扩展无需修改代码。每条规则有唯一的 rule_id、目标 section、触发时机 trigger、检查类型 check、严重级别 severity（error/warning/info）。

三层校验体系

层级触发时机检查内容
字段级onBlur 失焦数据类型、范围、格式
段落/章节级Next 点击 / 章节完成必填字段完整度、引用存在性、字数范围
全文级导出前引用-文献匹配、章节字数占比、标题非空

检查类型包括：required_field、required_fields、min_citations、word_count_range、citation_refs_match、figure_refs_exist、table_refs_exist、section_word_ratio。 每种检查类型对应 _check_rule() 中的一个分支逻辑。

    # Step 2: 匹配 @type{key, fields...}
    entry_pattern = re.compile(
        r"@(\w+)\s*\{\s*([^,]+)\s*,\s*(.+?)\}\s*$",
        re.DOTALL | re.MULTILINE)

    for match in entry_pattern.finditer(raw):
        entry_type = match.group(1).lower()   # article|book|...
        cite_key   = match.group(2).strip()   # smith2024
        fields_str = match.group(3)            # title={...}, author={...}

        # Step 3: 解析 key = {value} 键值对
        fields = cls._parse_bibtex_fields(fields_str)
        # Step 4: "Smith, John and Doe, Jane" → [{first, last}]
        authors = cls._parse_author_string(fields["author"])

RIS 解析流程

# reference_importer.py
@classmethod
def _parse_ris(cls, raw: str) -> ImportResult:
    for line in raw.splitlines():
        # Step 1: 匹配双字母标签 "TY  - value"
        match = re.match(r"^(\w{2})\s{2}-\s+(.*)$", line)

        tag, value = match.group(1), match.group(2).strip()

        if tag == "ER":     # End of Record
            entries.append(cls._build_ris_entry(current))
        elif tag == "TY":   # Start of new record
            current = {"type_tag": value}
        elif tag in ("AU", "KW"):
            # Multi-value fields
            current[tag].append(value)

# Step 2: 构建结构化引用对象
cls._build_ris_entry(raw):
    return {
        "ref_type": cls.RIS_TYPE_MAP[type_tag],
        "title": raw["TI"],
        "authors": cls._parse_author_string(
            "; ".join(raw["AU"])),
        "journal": raw["JO"], ...
    }

作者名解析：两种格式自动识别

@staticmethod
def _parse_author_string(author_str: str) -> List[Dict[str, str]]:
    # Split on "and" (BibTeX convention)
    names = re.split(r"\s+and\s+", author_str, flags=re.IGNORECASE)

    for name in names:
        if "," in name:
            # "Smith, John" → first="John", last="Smith"
            last, first = name.split(",", 1)
        else:
            # "John Smith" → first="John", last="Smith"
            parts = name.split()
            first, last = " ".join(parts[:-1]), parts[-1]
        authors.append({"first": first.strip(), "last": last.strip()})

    return authors

同时支持 detect_format() 自动识别输入是 BibTeX 还是 RIS 格式（根据是否包含 @ 开头条目或 TY 标签），用户无需手动指定。

11 · 技术创新点

6 个值得评审关注的创新设计

🌳

1. Document AST 中间表示

设计了一个完整的文档对象模型（AST），将论文表示为 Section → Paragraph → Run（TextRun / CitationRun / FigureRef / EquationRef / TableRef）的树结构。 这是整个系统架构的核心创新。所有导出格式共享同一编译管线，新增导出格式只需实现一个新的 Compiler 类。

🧩

2. 带类型系统的插槽式模板引擎

不是简单的字符串替换。每个插槽都有类型定义（text / number / select）、校验规则、选项列表和提示文本。 PhraseAssembler.fill() 方法在填充前执行 _validate()：检查必填项、验证 number 类型可转换、确认 select 值在预设选项中。 校验失败抛出 SlotValidationError 并精确报告问题插槽。

📋

3. 声明式校验规则引擎

12 条校验规则以纯 JSON 对象数组存储——可以放进数据库由用户自定义，不需要修改任何代码。 每条规则有 7 个字段（rule_id / section / trigger / check / field / severity / message / auto_fix_hint）， 支持 6 种 check_type。三级触发（on_blur / on_section_complete / on_export）精确控制校验时机。

📐

4. 5 种引文格式的纯 Python 实现

CitationFormatter 用 319 行代码实现了 5 种完整引文格式规范——IEEE（方括号编号）、APA 7th（作者-年份括号）、 Vancouver（上标编号）、Nature（紧凑上标）、MLA 9th（作者-页码体例）。 每种格式都有独立的作者名规范化函数（缩写规则不同），可以基于字段组合的 JSON 模板配置任意新格式。

🖋️

5. 手写 BibTeX/RIS 双格式解析器

用正则表达式从零实现了 BibTeX（@article{key, field={...}}）和 RIS（双字母标签语法）两种文献格式的解析器。 支持嵌套花括号值、多值字段聚合、作者名 "Smith, John" 和 "John Smith" 两种格式的自动识别与标准化。 附带 detect_format() 自动检测功能——根据文本中是否含有 @ 开头的条目或 TY 标签来判断格式。

🎯

6. 全链路 MVP 闭环

从创建论文 → 分步填写 → 句型库辅助 → 文献导入 → 交叉引用 → 校验 → LaTeX/PDF/Word 导出，全链路跑通。 62 个文件、6,904 行代码（含前端），前后端分离架构，Docker 一键部署（含 LanguageTool 语法检查服务）。 数据持久化采用 PostgreSQL JSONB —— 保留结构化段落信息的同时支持 SQL 查询。
