普通人写 SCI 论文的三座大山

🧱

结构壁垒

SCI 论文要求严格遵守 IMRaD 结构，但普通人打开空白文档后完全不知道该写什么。Introduction 的三步论证、Discussion 的五步推演——这些学术界默认的"隐知识"，普通人无从获取。

🗣️

语言壁垒

学术英语是一套独立的修辞系统。"a significant increase was observed" 和 "things went up" 表达同一事实，审稿人只接受前者。问题不在语法——在于不知道有哪些表达范式可选。

📐

工具壁垒

LaTeX 学习曲线陡峭。不同期刊对字体、行距、图表规范、引文格式有完全不同且严格的要求。手动管理编号和交叉引用极易出错——对非计算机背景的人来说几乎不可能。

本系统的核心理念是"不替代用户思考，而是提供结构和选项"。我们拒绝使用任何 AI 文本生成技术——用户始终是自己论文的唯一作者。系统提供三层辅助：结构引导、语言选项、格式自动化。每一层的输出都是确定性的——相同输入产生相同输出。

序列化工厂

每个 AST 节点有 from_dict() 和 to_dict()。JSONB 中的 "type" 字段作为分发器。

# Run 反序列化工厂
_RUN_TYPE_MAP = {
    "text": TextRun,
    "citation": CitationRun,
    "figure_ref": FigureRef,
    ...
}
def run_from_dict(d):
    return _RUN_TYPE_MAP[
        d["type"]
    ].from_dict(d)

六大核心引擎

1. BibTeX/RIS 解析器

手写实现，零外部依赖。括号深度追踪算法处理嵌套花括号值、作者名双格式自动识别（"Smith, John" / "John Smith"）、姓氏粒子检测（van/der/von）、后缀识别（Jr./III）。

2. 引文格式化引擎

五种风格独立实现：IEEE（方括号+缩写名）、APA 7th（作者-年份+斜体期刊）、Vancouver（上标编号）、Nature（紧凑上标）、MLA 9th（作者-页码）。每种风格有独立作者名规范化函数。

3. LaTeX 编译器

DocumentAST → 完整 .tex 文件。内置 4 种期刊 preamble，10 个特殊字符转义映射。逐段编译：TextRun → 转义文本，CitationRun → \cite{}。双次 xelatex 编译生成 PDF。

4. Word 编译器

AST → python-docx → .docx。格式规则配置：字体族、字号、行距（单倍/1.5/双倍）、页边距。交叉引用转换为数字格式。

5. 自动编号引擎

Sequential（Fig. 1, 2, 3...）和 Chapter-based（Fig. 2.1, 2.2...）双模式。遍历元素按章节排序后分配编号并生成交叉引用标签。

6. 声明式校验引擎

14 条规则以 JSON 对象数组存储。7 种检查类型：required_field / required_fields / word_count_range / min_citations / citation_refs_match / figure_refs_exist / table_refs_exist。三级触发：字段失焦 → 步骤完成 → 导出前。
