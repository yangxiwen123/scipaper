🎯 要解决的问题

普通人写 SCI 论文的三座大山

传统 SCI 论文写作对非学术背景的人来说极其困难。我们的系统通过纯软件工程手段逐一攻克。

🧱

结构壁垒：不知道每部分写什么

6 步 Wizard 向导将 IMRaD 结构拆解为填空题和下拉选择。每个文本框旁有 [ ? ] 图标，说明该段落在 SCI 论文中的功能、推荐长度、示范句型。

🗣️

语言壁垒：不会写"学术英语"

内置 30 条精心编排的学术句型模板（基于 Manchester Phrasebank 分类体系），覆盖 Introduction / Methods / Results / Discussion / Conclusion 所有章节。按功能分类树浏览，一键插入，变量插槽填充。

📐

排版壁垒：搞不定 LaTeX 和期刊格式

结构化 JSON → Document AST → LaTeX / Word 自动编译管线。预置 IEEE、Elsevier、Springer、Nature 四种期刊模板，图表公式自动编号，5 种引文格式一键切换。

📚 学术句型库 (Phrasebank)

动态模板引擎，不是文本生成

我们不依赖任何 AI 生成技术。每条句型都是手工编排的模板，用户通过插槽变量填充来完成学术表达。

# 句型模板存储（数据库 seed）
{
  "section": "introduction",
  "function_tag": "identifying_gap",
  "template_text": "Despite considerable research on {existing_work},
                        {gap_description} remains poorly understood.",
  "slots": {
    "existing_work": {"type": "text", "hint": "what has been studied"},
    "gap_description": {"type": "text", "hint": "what we don't know yet"}
  },
  "example_filled": "Despite considerable research on antibiotic resistance mechanisms,
                        the role of environmental factors in resistance gene transfer remains poorly understood."
}

# 用户填充后 → 编译为论文段落

Document AST 中间表示

论文数据在数据库中以结构化 JSON 存储，导出时先编译为统一的 Document AST（段落、引用、图表、公式的文档树），再分别编译到 LaTeX 或 Word。

🔢

自动编号引擎

NumberingEngine 为全文的 Figures、Tables、Equations 自动分配连续编号（支持全文顺序编号和按章节编号两种模式），导出时自动生成交叉引用链接。

✅

三层校验体系

字段级（onBlur 即时反馈）→ 段落级（Next 步骤时完整性检查）→ 全文级（导出前图表引用、参考文献、字数占比检查）。12 条内置规则。

🚀 一键启动

Demo 运行方式

🐳

Docker 全栈启动

cd D:\CodeProject\scipaper
docker-compose up -d
docker-compose exec backend python -m seed_data.run_seed

启动 PostgreSQL + Redis + LanguageTool + Backend + Celery，然后初始化种子数据。API 文档自动生成在 http://localhost:8000/docs

💻

本地开发模式

cd backend && uvicorn app.main:app --reload
cd frontend && npm install && npm run dev

后端需要本地 PostgreSQL 或修改 config.py 使用 SQLite。前端自动代理到 :8000

💡 技术创新点

为什么这个项目有创造力

🔀

Document AST 中间表示

设计了一个完整的文档对象模型（AST），将论文内容表示为 Section → Paragraph → Run（TextRun / CitationRun / FigureRef / EquationRef）的树结构。所有导出格式共享同一编译管线，新增格式只需写一个 Compiler。

🧩

插槽式模板引擎

不是简单的字符串替换——每个插槽都有类型定义（text / number / select）、校验规则、选项列表和提示文本。模板装配器能验证用户填入的值是否符合类型预期，并返回缺失插槽列表。

📋

声明式校验规则引擎

12 条校验规则以 JSON 对象形式存储（可扩展为数据库行），支持 required_field / word_count_range / min_citations / citation_refs_match 等检查类型。三级触发机制（字段失焦 / 步骤完成 / 全文导出前）。

📐

引用格式系统

CitationFormatter 用纯 Python 实现了 5 种主流学术引文格式的规范输出——IEEE 的方括号编号、APA 的作者-年份括号、Vancouver 的上标编号、Nature 的紧凑格式、MLA 的作者-页码体例。每种格式都有独立的作者名规范化逻辑。

🖋️

手写解析器

ReferenceImporter 用正则表达式从零实现了 BibTeX 和 RIS 两种文献格式的解析器——而非依赖第三方库。支持 BibTeX 嵌套花括号值、RIS 双字母标签语法、作者名字段 "Smith, John" 和 "John Smith" 两种格式的自动识别。

🎨

完整的 MVP 闭环

从创建论文 → 分步填写 → 句型库辅助 → 文献导入 → 交叉引用 → 校验 → LaTeX/PDF/Word 导出，全链路跑通。60 个文件、6904 行代码，前后端分离，Docker 一键部署。
